# 多端购物商城产品规划（v0.1.0）

## 1. 产品目标
构建“可交付演示级”多端购物商城：覆盖 Web、Windows、macOS、iOS、Android、鸿蒙、微信小程序。统一业务能力，按端适配 UI 与交互，支持快速从 Mock 迁移到真实后端。

## 2. 用户角色
- 游客：浏览、搜索、查看商品详情
- 注册用户：下单、支付、评价、收藏、地址管理、售后
- 商家运营：商品管理、库存管理、订单处理、活动配置
- 平台运营：类目与内容管理、风控、数据看板、客服管理

## 3. 核心功能（主流商城）
- 商品域：类目、搜索、筛选、详情、SKU、库存、评价
- 交易域：购物车、下单、优惠券、支付、订单生命周期
- 用户域：注册登录、个人中心、地址、收藏、浏览历史
- 营销域：秒杀、满减、推荐位、Banner、限时活动
- 履约域：物流跟踪、签收、售后、退款

## 4. 页面信息架构（IA）
- 首页：Banner、频道入口、推荐商品、活动楼层
- 分类页：一级/二级类目、筛选面板
- 搜索页：热词、联想、结果列表
- 商品详情：图文、规格、评价、推荐、加入购物车/立即购买
- 购物车：SKU 列表、数量编辑、结算
- 结算页：地址、配送、优惠、支付方式
- 订单页：订单列表、详情、物流、售后入口
- 我的：账户资料、地址、收藏、优惠券、设置

## 5. MVP 范围（本次交付）
- 多端基础壳工程（Web + Taro 小程序/移动 + Tauri 桌面）
- 商品列表/详情、购物车、下单流程（模拟支付）
- 用户中心基础页面
- Mock 数据 + OpenAPI 契约 + MSW 拦截
- 响应式 UI（PC + 移动）
- 测试体系：单测、组件测试、E2E（Web）

## 6. 后续迭代
- v0.2：账号体系（短信/第三方登录）、优惠券核销、订单状态机强化
- v0.3：运营后台（商家与平台管理）、埋点分析、A/B 实验
- v0.4：推荐系统、直播带货、智能客服、风控引擎
